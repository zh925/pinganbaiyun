# ADR-003：门禁密钥本地保护与备份策略

- 状态：提议
- 日期：2026-07-17
- 决策人：待用户确认
- 关联 Issue/PR：RUO-4、RUO-5、RUO-7；最终需求 PR 待创建

## 背景

门禁 key 是可产生物理访问后果的敏感凭据。`SafeBaiyun` 使用普通 SharedPreferences 且允许系统备份；RUO-4 的全新实现不得照搬。单 DES 和静态设备 key 仍是设备协议遗留风险，本地加密不能消除链路或设备端风险。

## 选项

1. Keystore 支持的加密存储、禁止系统备份、无导入导出；UI/日志全面脱敏。
2. 允许系统备份/设备迁移；使用方便但扩大凭据副本和恢复面。
3. 不持久化 key，每次输入；最安全但不满足默认门禁自动开门。

## 决策

提议选择选项 1。最终决定留给用户。

## 理由

- 满足离线与自动开门的持久化需要，同时减少明文和备份暴露。
- 不回退到 SafeBaiyun 的明文存储，并为全新实现建立最低保护基线。
- 禁止导出可避免把物理访问凭据带入剪贴板、文件和分享渠道。

## 影响与后续

- key 默认不可见、不可直接复制；编辑时不在日志或辅助文本泄漏。
- 应用卸载、Keystore 失效或设备迁移后可能无法恢复，用户需重新录入。
- 解密失败不得静默清空或覆盖原密文，必须提示并经确认后重置。
- debug/release 日志、崩溃报告、测试夹具和 Issue 证据均不得包含真实 key、完整 seed 或完整可重放帧。
- 需要安全测试验证备份禁用、私有存储无明文和删除后的逻辑不可访问；不承诺闪存物理安全擦除。
