# 交互与验收映射

| 原型入口/状态 | 关键行为 | 对应验收 |
|---|---|---|
| 门禁列表 | 本地加载、MAC/key 脱敏、手动开门 | AC-02、AC-13、AC-34、NFR-04/05 |
| 空状态 | 不扫描、不连接、不请求权限/网络 | AC-01、AC-27 |
| 新增/编辑弹层 | 必填、MAC、16 位 HEX key、重复 MAC 校验；编辑保持默认关系 | AC-02–05 |
| 管理门禁弹层 | 设置、切换、取消唯一默认门禁 | AC-08–10、NFR-06 |
| 删除确认 | 默认门禁同步取消且不改选 | AC-06、AC-07 |
| 模拟冷启动 | 有默认时最多一次；无默认时无 BLE 任务 | AC-26–29、AC-35 |
| 开门中 | 已知 MAC 直连；连接→读 seed→发送→确认写入；全局单任务 | AC-14–15、AC-20、NFR-01/02 |
| 模拟前后台 | 恢复同一 taskId，不重复触发 | AC-28–30 |
| 取消 | 清理任务；迟到回调不改写终态 | AC-21、AC-22、NFR-11 |
| 指令已发送 | 仅确认 GATT 写成功，不宣称门已打开 | AC-18 |
| 设备不可达/超时 | 唯一失败终态、无自动重试、可显式重试 | AC-19、AC-23–25、NFR-03 |
| 权限缺失/蓝牙关闭 | 未就绪不发起 GATT；授权/开启后只继续原任务 | AC-28、AC-31–33 |
| 数据不可读取 | 不静默覆盖，安全重置需再次确认 | AC-12、NFR-07 |
| 响应式/减弱动态 | 窄屏单栏、状态不只依赖颜色、按钮有文本 | NFR-08/09 |

## 非原型验证项

- AC-11、AC-16、AC-36–39 与 NFR-01–08、NFR-10–12 中涉及真实持久化、协议字节、Android 生命周期、构建、日志、备份、性能、资源释放和真机兼容性的结论，必须在开发与获授权真机阶段验证；浏览器原型不宣称通过。
- AC-17 的用户可见协议失败终态由“协议失败”场景覆盖；seed 长度、DES 构造和“不发送部分帧”只能由后续单元测试与假 GATT 验证。
- AC-30 的任务快照与后续重试规则已在 README 记录，原型进度页仅展示“当前任务使用启动时配置快照”，真实并发行为留待 Android 仪器测试。
- AC-40 是最终需求基线人工门禁，不由本原型代替确认。

## 协议表达边界

- 原型不出现“扫描中”，因为按 ADR-001 待最终确认的推荐选项展示已知 MAC 直连。
- 原型把 `onCharacteristicWrite(GATT_SUCCESS)` 表述为“指令已发送”，没有设备回执证据时不使用“开门成功”。
- 原型不会呈现 key、seed、DES 密文或完整可重放帧；演示 MAC 也做了局部掩码。
- `SafeBaiyun` 20 字节帧、Service/Characteristic、DES 与异步 GATT 细节仅由 Stage 1 文档定义，原型不新增协议结论。
