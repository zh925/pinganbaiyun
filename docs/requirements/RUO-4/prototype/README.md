## 页面与流程

- 门禁列表：展示本机加密存储提示、唯一默认门禁、MAC 脱敏、手动开门和管理入口。
- 空状态与加载：首次使用不发起 BLE；读取本机加密数据时明确“不请求网络”。
- 新增/编辑：名称、MAC、恰好 16 位 HEX key 就地校验；重复 MAC 被拒绝；key 使用密码输入且不提供复制。
- 删除：始终二次确认；删除默认门禁时明确同步取消默认且不自动改选。
- 默认门禁：支持设置、切换、取消；工具栏“模拟冷启动”仅在存在默认门禁时创建一次自动任务。
- 手动/自动开门：准备后依次展示连接设备、读取种子、发送指令、确认写入。原型按待最终确认的推荐基线展示已知 MAC 直连，不展示 BLE 扫描。
- 结果：GATT 写成功使用“开门指令已发送，请确认门禁的实际状态”，不声称物理门已打开。
- 异常：场景选择器覆盖加载、空、设备不可达、超时、权限缺失、蓝牙关闭和本地数据损坏。

## 交互规则

- 任一时刻只允许一个开门任务。进行中连续点击或冷启动触发不会创建第二个任务。
- 进程级 Launcher 冷启动可自动触发一次；热启动、旋转、回前台及权限/蓝牙页返回只恢复原 taskId。
- 自动或手动失败均不自动重试；“重试”显式创建新任务并读取最新保存配置。
- 非终态可取消；取消清理任务并隔离迟到回调。
- 设置新默认门禁时原子替换旧默认；取消或删除默认后，后续冷启动不自动开门。
- 编辑正在执行的配置不改变任务快照；原型通过进度页说明该约束，不模拟真实 GATT。
- 详细验收映射见 [interaction-map.md](interaction-map.md)。

## 页面状态

- 加载：本机数据解密加载态，说明无网络请求。
- 空状态：无门禁时仅显示新增入口，不请求权限、不扫描、不连接。
- 错误：设备不可达、超时、协议/数据异常均进入唯一终态，并给出返回或显式重试。
- 权限不足：区分附近设备权限与蓝牙关闭；拒绝后不循环申请，系统页返回只继续原任务。
- 响应式：桌面为“设计说明 + 设备画布”；≤900px 收敛为上下布局；≤430px 使用全宽移动端布局；支持 `prefers-reduced-motion`。
